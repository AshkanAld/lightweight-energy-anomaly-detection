# Lightweight Energy Anomaly Detection

A reproducible Python project for detecting simulated faults and cyber-attacks in cyber-physical energy systems using lightweight machine-learning methods.

## Project Motivation

Modern cyber-physical energy systems, including power converters, electric-vehicle chargers, smart-grid components, and industrial electrical infrastructure, rely on sensor measurements and embedded control platforms.

Faulty sensors, abnormal operating conditions, and manipulated measurements may lead to unreliable control decisions or physical damage. This project investigates lightweight machine-learning methods for identifying such conditions.

## Current Project Scope

The first version of the project uses a synthetically generated dataset representing normal and anomalous operating conditions.

The simulated measurements include:

- AC voltage
- Current
- Grid frequency
- Equipment temperature
- DC-link voltage
- Total harmonic distortion
- Active power

The simulated anomaly scenarios include:

- Voltage sag
- Overvoltage
- Current spike
- Temperature anomaly
- Sensor-bias attack
- False-data injection

## Model Comparison

Three lightweight anomaly-detection approaches were evaluated on the synthetic test set.

| Model | Accuracy | Precision | Recall | F1-score | ROC-AUC | False-negative rate | Model size |
|---|---:|---:|---:|---:|---:|---:|---:|
| Logistic Regression | 89.4% | 71.8% | 77.5% | 74.5% | 0.909 | 22.5% | 1.88 KB |
| Random Forest | 100% | 100% | 100% | 100% | 1.000 | 0% | 1063.70 KB |
| Isolation Forest | 97.5% | 95.3% | 92.0% | 93.6% | 0.997 | 8.0% | 1478.09 KB |

Random Forest achieved the best predictive performance, while Logistic Regression was by far the smallest and fastest model. Isolation Forest provided strong anomaly-detection performance without using class labels during training.

The perfect Random Forest result likely reflects the clear separation of the simulated anomaly patterns. These results should therefore be interpreted as a proof of concept rather than evidence of equivalent performance on real-world cyber-physical energy-system data.

## Deployment-Oriented Analysis

The models were compared in terms of confusion-matrix performance, feature importance, inference time, and serialized model size.

### Confusion Matrix Summary

| Model | True Negative | False Positive | False Negative | True Positive |
|---|---:|---:|---:|---:|
| Logistic Regression | 739 | 61 | 45 | 155 |
| Random Forest | 800 | 0 | 0 | 200 |
| Isolation Forest | 791 | 9 | 16 | 184 |

Random Forest achieved zero false negatives on the current synthetic test set. Isolation Forest missed 16 anomalous observations, while Logistic Regression missed 45.

### Inference Time and Model Size

| Model | Median inference time | Model size |
|---|---:|---:|
| Logistic Regression | 0.002739 ms/sample | 1.88 KB |
| Random Forest | 0.101078 ms/sample | 1063.70 KB |
| Isolation Forest | 0.068670 ms/sample | 1478.09 KB |

Logistic Regression was by far the smallest and fastest model. Random Forest provided the strongest detection performance but required substantially more storage and inference time.

Inference times were measured on the author's local computer and are intended for relative comparison rather than hardware-independent benchmarking.

## Limitations

This project is a proof of concept and has several important limitations:

1. **Synthetic data only**  
   The models were trained and evaluated on synthetically generated energy-system measurements. The simulated data do not fully represent the complexity, noise, operating transitions, and uncertainties of real cyber-physical energy systems.

2. **Clearly separated anomaly patterns**  
   Several anomaly types were generated using relatively strong deviations from normal operating conditions. This may explain the perfect Random Forest performance and can make the classification task easier than real-world fault or attack detection.

3. **Limited operating conditions**  
   The dataset represents a simplified operating regime and does not include changing loads, converter-switching dynamics, seasonal variation, startup conditions, shutdown conditions, or multiple interacting devices.

4. **Simplified attack scenarios**  
   The simulated sensor-bias and false-data-injection attacks are basic representations. Real attacks may be gradual, coordinated, adaptive, stealthy, or designed specifically to bypass detection systems.

5. **No independent external validation**  
   The models were evaluated using a train-test split generated by the same synthetic-data process. They have not yet been tested on an independent real-world or hardware-generated dataset.

6. **No hardware deployment**  
   Inference time was measured on a local computer. The reported latency and model size provide only an initial indication of deployment suitability and do not demonstrate performance on a microcontroller, FPGA, embedded processor, or edge device.

7. **Limited hyperparameter optimization**  
   The models used reasonable predefined configurations, but no systematic cross-validation or extensive hyperparameter optimization was performed.

8. **Correlated input features**  
   Active power is derived from voltage and current measurements and therefore contains correlated information. This can affect coefficient interpretation and feature-importance analysis.

9. **Known anomaly proportion for Isolation Forest**  
   The Isolation Forest contamination parameter was set to 0.20, matching the simulated anomaly proportion. In a real application, the true anomaly rate may be unknown and could vary over time.

10. **Binary detection only**  
    The current models classify observations only as normal or anomalous. They do not directly identify the specific fault or cyberattack category.

## Future Work

Future development may include:

- validation using real or hardware-in-the-loop energy-system data;
- generation of more subtle and overlapping anomaly patterns;
- multiclass identification of individual fault and attack types;
- time-series and sequential anomaly detection;
- threshold optimization using a separate validation set;
- comparison with lightweight neural-network models;
- model compression, pruning, and quantization;
- deployment on embedded processors or edge devices;
- evaluation under concept drift and changing operating conditions;
- investigation of adversarial robustness and secure inference.

### Feature Importance

Random Forest identified voltage, temperature, total harmonic distortion, and current as the most informative features. Logistic Regression relied most strongly on current and active power.

Active power is derived from voltage and current measurements and therefore introduces correlated information into the feature set.

Clone the repository:

```bash
git clone https://github.com/YOUR-USERNAME/lightweight-energy-anomaly-detection.git

## Usage

Run all commands from the root directory of the repository.

Generate the synthetic energy-system dataset:

```bash
python src/generate_data.py
```

Validate, clean, and split the dataset:

```bash
python src/preprocess.py
```

Generate exploratory data visualizations:

```bash
python src/visualize.py
```

Train and evaluate Logistic Regression:

```bash
python src/train_logistic_regression.py
```

Train and evaluate Random Forest:

```bash
python src/train_random_forest.py
```

Train and evaluate Isolation Forest:

```bash
python src/train_isolation_forest.py
```

Compare the models:

```bash
python src/compare_models.py
```

Run the deployment-oriented analysis:

```bash
python src/analyze_models.py
```
